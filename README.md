# 🎬 TikTok Views Booster API

API Backend آمن لخدمة رشق مشاهدات تيك توك، مع إخفاء مفتاح API عن المستخدمين.

## 📋 المميزات

- ✅ **API آمن:** مفتاح API مخفي على الخادم
- ✅ **CORS مفعّل:** يعمل مع أي موقع ويب
- ✅ **سهل الاستضافة:** جاهز للاستضافة المجانية على Render أو Railway
- ✅ **معالجة أخطاء:** رسائل خطأ واضحة وآمنة
- ✅ **JSON Response:** ردود بصيغة JSON

---

## 🚀 البدء السريع

### المتطلبات
- Node.js 14+ ([تحميل من هنا](https://nodejs.org/))
- npm (يأتي مع Node.js)

### التثبيت المحلي

```bash
# 1. استنساخ المشروع أو تحميل الملفات
cd tiktok-api

# 2. تثبيت الحزم
npm install

# 3. إنشاء ملف .env (اختياري)
cp .env.example .env

# 4. تشغيل الخادم
npm start
```

الخادم سيعمل على `http://localhost:3000`

---

## 📡 نقاط النهاية (Endpoints)

### 1. فحص حالة الخادم
```
GET /api/health
```
**الرد:**
```json
{"status":"ok","message":"Server is running normally"}
```

### 2. إنشاء طلب جديد
```
POST /api/order
Content-Type: application/json

{
  "link": "https://www.tiktok.com/@user/video/123456789"
}
```

**الرد (نجاح):**
```json
{"success":true,"data":{...}}
```

**الرد (خطأ):**
```json
{"success":false,"error":"الرابط مطلوب"}
```

## 🔐 الأمان

- **مفتاح API:** مخفي في متغيرات البيئة
- **CORS:** مفعّل للسماح بالطلبات من أي دومين
- **معالجة الأخطاء:** لا تكشف معلومات حساسة

## 🐛 استكشاف الأخطاء

| المشكلة | الحل |
|--------|------|
| "Port already in use" | غيّر PORT في .env أو أغلق التطبيق الآخر |

| "API not responding" | تأكد من اتصال الإنترنت والمفتاح صحيح |
