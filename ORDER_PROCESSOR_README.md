# Order Processor - نظام معالجة الطلبات المتقدم

## 📋 نظرة عامة

نظام متقدم لمعالجة الطلبات من لوحة التحكم الإدارية مع نظام فلترة ذكي يكتشف الطلبات المكررة ويمنع الاحتيال.

## 🎯 المميزات الرئيسية

### 1. **فحص الطلبات المعلقة**
- يفحص فقط الطلبات ذات حالة `Pending`
- يركز على الخدمة رقم **17337** فقط
- يتحقق كل 5 دقائق تلقائياً

### 2. **نظام الفلترة المتقدم**

#### أ) فحص المستخدم (User Duplicate Check)
- يتحقق إذا قام المستخدم بطلب خلال آخر **30 دقيقة**
- إذا كان هناك طلب سابق → **إلغاء الطلب مع استرجاع المبلغ** (Cancel and Refund)
- يستخدم hash آمن لتعريف المستخدم

#### ب) فحص الرابط (URL Duplicate Check)
- يتحقق إذا تم استخدام نفس الرابط خلال آخر **30 دقيقة**
- تنظيف الرابط من الشوائب:
  - إزالة البروتوكول (http://, https://)
  - إزالة www
  - إزالة المعاملات (query parameters)
  - إزالة الـ fragments
- إذا كان هناك طلب سابق → **إلغاء الطلب مع استرجاع المبلغ**

### 3. **إعادة توجيه الطلبات**
- إذا مرّ الطلب جميع الفحوصات → **إرسال للمرحلة الثانية**
- يتم إرسال الطلب إلى: `https://kd1s.com/apikd1s`
- يستخدم المفتاح: `ce5d33dc71b144c60cab2f8f977bbc21`
- يتم تحويل الخدمة من **17337** إلى **17828**
- يتم الحفاظ على نفس الكمية

## 📁 ملفات البيانات

### 1. `processed_orders.json`
- يحتفظ بسجل الطلبات المعالجة
- يمنع معالجة نفس الطلب مرتين

### 2. `user_orders.json`
- يتتبع آخر طلب لكل مستخدم
- يستخدم hash المستخدم كمفتاح
- يحتفظ بـ timestamp آخر طلب

### 3. `url_orders.json`
- يتتبع آخر استخدام لكل رابط
- يستخدم hash الرابط المنظف كمفتاح
- يحتفظ بـ timestamp آخر استخدام

## 🔐 الأمان

### تجزئة البيانات (Hashing)
- يتم تجزئة معرفات المستخدمين باستخدام MD5
- لا يتم تخزين بيانات المستخدمين الحساسة

### تنظيف الروابط (URL Normalization)
- إزالة البروتوكولات
- إزالة المعاملات والـ fragments
- تحويل إلى lowercase
- هذا يضمن اكتشاف الروابط المكررة حتى لو كانت بصيغ مختلفة

### معالجة الأخطاء
- محاولات إعادة في حالة الفشل
- تسجيل شامل للأخطاء
- timeout محدد لكل طلب API

## 📈 الأداء

- **معالجة الطلبات**: كل 5 دقائق تلقائياً
- **تأخير بين الطلبات**: 1 ثانية (لتجنب Rate Limiting)
- **Timeout للطلبات**: 30 ثانية

## 🐛 استكشاف الأخطاء

### الطلب لم يتم معالجته
- تحقق من حالة الطلب (يجب أن تكون `Pending`)
- تحقق من رقم الخدمة (يجب أن تكون 17337)
- تحقق من سجلات الخادم

### الطلب تم إلغاؤه بدون سبب
- قد يكون المستخدم قد أرسل طلب قبل 30 دقيقة
- قد يكون الرابط تم استخدامه قبل 30 دقيقة
- تحقق من ملفات البيانات

### الفشل في الإرسال للمرحلة الثانية
- تحقق من الاتصال بالإنترنت
- تحقق من صحة المفتاح API
- تحقق من صحة رقم الخدمة 17828
